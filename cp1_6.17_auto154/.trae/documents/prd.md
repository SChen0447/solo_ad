## 1. 产品概述

团队工单看板（Team Ticket Board）是一款面向小型团队的内部工单反馈收集与可视化管理工具，解决团队成员通过邮件、即时消息、口头等多渠道报告问题时信息散落、难以追踪和量化工作负荷的痛点。通过看板式工单管理和数据统计面板，实现工单的集中化、可视化和可量化管理。

- 目标用户：5-20人小型团队
- 核心价值：零门槛加入、拖拽式工单流转、实时统计可视化

## 2. 核心功能

### 2.1 用户角色

| 角色 | 加入方式 | 核心权限 |
|------|----------|----------|
| 团队成员 | 分享链接或6位团队码 | 查看、创建、拖拽工单 |

### 2.2 功能模块

1. **看板页面**：工单卡片墙（三列布局）、筛选栏、统计概览条、新建工单模态框
2. **统计面板页面**：柱状图（每日新增）、饼图（优先级占比）、折线图（人均处理数）

### 2.3 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|
| 看板页面 | 筛选栏 | 按优先级、提交人、时间范围实时过滤工单，0.2秒缓动过渡 |
| 看板页面 | 统计概览条 | 实时显示总工单数、各优先级数量、平均处理时长 |
| 看板页面 | 待处理列 | 展示状态为待处理的工单卡片，支持拖入/拖出 |
| 看板页面 | 进行中列 | 展示状态为进行中的工单卡片，支持拖入/拖出 |
| 看板页面 | 已完成列 | 展示状态为已完成的工单卡片，支持拖入/拖出 |
| 看板页面 | 新建工单模态框 | 表单含标题、描述（200字限制）、优先级选择；提交后卡片淡入上移至待处理列顶部 |
| 工单卡片 | 卡片主体 | 显示标题、优先级色条、提交人、简短描述、提交时间 |
| 工单卡片 | 关联引用 | 描述中@T-XXX引用其他工单，右侧显示关联数量徽章 |
| 统计面板 | 柱状图 | 近7天每日新增工单数，鼠标悬停显示具体数值 |
| 统计面板 | 饼图 | 各优先级占比，鼠标悬停显示具体数值 |
| 统计面板 | 折线图 | 团队人均工单处理数变化趋势，鼠标悬停显示具体数值 |

## 3. 核心流程

1. 用户通过分享链接或6位团队码进入团队看板
2. 看板以三列卡片墙展示所有工单（待处理/进行中/已完成）
3. 用户点击"新建工单"按钮，填写标题、描述、优先级后提交
4. 工单自动出现在待处理列顶部，带淡入上移动画
5. 团队成员将工单卡片拖拽至不同状态列，更新工单状态
6. 拖拽时卡片轻微旋转（最大5度）并有弹性动画（0.3秒），松手后吸附到列内
7. 用户可通过筛选栏按优先级、提交人、时间范围过滤工单
8. 顶部统计概览条实时更新工单总数、优先级分布和平均处理时长
9. 用户切换到统计面板查看可视化报表

## 4. 用户界面设计

### 4.1 设计风格

- 主背景色：#F8F9FA（浅灰白）
- 卡片背景色：#FFFFFF（纯白）
- 文字色：#2C3E50（深蓝灰）
- 优先级色条：紧急#E74C3C、高#F39C12、中#3498DB、低#95A5A6
- 状态列背景：待处理#E8E8E8、进行中#D5E8D4、已完成#D4EFDF
- 字体：系统默认无衬线字体
- 按钮风格：圆角、扁平化、带hover阴影
- 布局风格：顶部导航+三列看板/统计面板
- 动画：平滑过渡0.2-0.5秒，拖拽旋转+弹性，新建淡入上移

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI元素 |
|----------|----------|--------|
| 看板页面 | 顶部导航栏 | 团队名称、团队码、看板/统计切换按钮、新建工单按钮 |
| 看板页面 | 筛选栏 | 优先级按钮组、提交人下拉、时间范围选择器 |
| 看板页面 | 统计概览条 | 总数/紧急/高/中/低数量标签、平均处理时长 |
| 看板页面 | 三列看板区 | 灰色列背景、卡片墙、拖拽目标区域 |
| 看板页面 | 新建工单模态框 | 居中弹窗、标题输入框、描述文本域（带字数计数）、优先级选择器、提交/取消按钮 |
| 统计面板 | 柱状图区 | 近7天柱状图，悬停tooltip |
| 统计面板 | 饼图区 | 优先级占比饼图，悬停tooltip |
| 统计面板 | 折线图区 | 人均处理数折线图，悬停tooltip |

### 4.3 响应式设计

- 桌面端优先（≥768px）：三列水平并排
- 移动端（<768px）：三列垂直堆叠，卡片全宽
- 拖拽在移动端通过长按触发
- 所有交互动画在移动端保持一致
