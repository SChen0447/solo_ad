## 1. 产品概述

宠物寄养家庭在线匹配与日程协调平台，解决宠物主人临时出差或旅行时找不到可靠寄养家庭，以及寄养家庭管理多只宠物日程混乱的痛点。

- 目标用户：宠物主人（寄养需求方）、社区宠物寄养家庭（寄养服务提供方）
- 核心价值：智能匹配推荐、可视化日程管理、实时同步沟通、信誉评价体系

## 2. 核心功能

### 2.1 用户角色

| 角色 | 说明 | 核心权限 |
|------|------|----------|
| 宠物主人 | 发布寄养需求的用户 | 发布需求单、查看推荐家庭、评价寄养家庭、实时查看照料记录 |
| 寄养家庭 | 提供寄养服务的用户 | 查看寄养任务、管理日程日历、记录照料事项、评价宠物主人 |

### 2.2 功能模块

1. **寄养需求发布与推荐**：需求表单、智能匹配算法、候选家庭推荐列表
2. **周视图日程管理**：日历展示、任务拖放、照料记录编辑、医疗提醒
3. **实时同步系统**：WebSocket双向通信、任务状态变更通知
4. **评价信誉系统**：星级评价、文字评价、信誉等级计算
5. **数据看板Dashboard**：整合所有子组件的主页面

### 2.3 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|
| Dashboard | 寄养需求表单 | 填写宠物品种、年龄、性格、寄养时间段、特殊护理需求 |
| Dashboard | 推荐家庭列表 | 显示匹配度百分比、家庭头像、评分、已服务宠物数 |
| Dashboard | 周视图日历 | 展示寄养任务、接送时间、喂食记录、医疗提醒 |
| Dashboard | 照料事项编辑 | 添加/修改散步时长、饮食要求、情绪状态记录 |
| Dashboard | 评价面板 | 寄养结束后1-5星评分和文字评价 |

## 3. 核心流程

宠物主人发布寄养需求后，系统根据寄养家庭的可用时间、地理位置和历史评价自动计算匹配度并生成推荐列表。主人选择寄养家庭后，寄养家庭在周视图日历中查看并管理该寄养任务，所有照料记录通过WebSocket实时同步给主人。寄养结束后，双方进行互评，系统更新信誉等级。

```mermaid

    A["宠物主人发布需求单"] --> B["系统计算匹配度"]
    B --> C["生成推荐家庭列表"]
    C --> D["主人选择寄养家庭"]
    D --> E["寄养任务进入日历"]
    E --> F["寄养家庭记录照料事项"]
    F --> G["WebSocket实时同步给主人"]
    G --> H["寄养结束双方互评"]
    H --> I["更新信誉等级与评分"]
```

## 4. 用户界面设计

### 4.1 设计风格

- **主色调**：暖橘色（#FF8C42）与奶油白（#FFF8F0）渐变，温暖自然
- **辅助色**：浅粉（#FFD6E0）、浅蓝（#D6E8FF）、浅绿（#D6F5E0）用于区分不同宠物任务
- **卡片风格**：毛茸茸大圆角（20px）、大片留白、柔和阴影
- **动效**：卡片悬停时轻微上浮+阴影加深（0.3s ease-out）、状态变更提示气泡淡入淡出（2s后消失）
- **字体**：圆润友好的无衬线字体，标题稍大加粗，正文舒适可读

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI元素 |
|----------|----------|--------|
| Dashboard | 需求表单 | 大圆角输入框、暖橘提交按钮、宠物性格标签选择 |
| Dashboard | 推荐列表 | 匹配度进度条、评分星星、家庭卡片悬停上浮动画 |
| Dashboard | 周视图日历 | 柔和色块区分宠物、可拖放任务块、医疗提醒图标 |
| Dashboard | 照料事项弹窗 | 情绪状态表情选择、散步时长滑块、饮食要求文本框 |
| Dashboard | 评价面板 | 5星交互评分、文字评价输入框、信誉等级徽章 |

### 4.3 响应式设计

- **桌面端**：三栏布局（左侧需求表单、中间推荐列表、右侧周视图日历）
- **平板端**：两栏布局，日历堆叠在下方
- **移动端**：单栏堆叠式布局，日历切换为任务列表，任务块支持左右滑动快速标记完成/延期

## 5. 性能要求

- 日历视图切换和任务编辑响应时间 ≤ 100ms
- 推荐列表生成与排序在筛选条件调整后 ≤ 300ms 完成渲染
- WebSocket消息延迟 ≤ 50ms
