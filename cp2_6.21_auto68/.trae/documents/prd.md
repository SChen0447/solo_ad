## 1. 产品概述
手工坊订单管理与生产排程应用，面向中小型手工作坊运营者，解决接单、采购物料、排产计划和发货之间的时间错配与库存积压问题，实现从接单到出货的全流程可视化。

## 2. 核心功能

### 2.1 用户角色
| 角色 | 注册方式 | 核心权限 |
|------|----------|----------|
| 作坊运营者 | 直接使用 | 订单管理、物料管理、工单排程、发货管理全权限 |

### 2.2 功能模块
1. **仪表盘页面**: 生产概览卡片（今日订单数、待开工工单数、库存预警数、待发货数）、工单列表、7天订单量/完工量折线图
2. **订单看板页面**: Kanban风格订单状态展示（待处理>生产中>待发货>已完成）、新建订单表单
3. **物料库存页面**: 物料表格（含库存预警高亮）、补货操作
4. **发货计划页面**: 按日期垂直时间线排列待发货订单、确认发货操作

### 2.3 页面详情
| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|
| 仪表盘 | 概览卡片 | 展示今日订单数、待开工工单数、库存预警数、待发货数 |
| 仪表盘 | 工单列表 | 展示工单信息（订单编号、时间、状态Badge），可点击查看详情Modal |
| 仪表盘 | 趋势折线图 | Canvas绘制最近7天订单量和完工量趋势，悬停显示tooltip |

| 订单看板 | 新建订单 | 点击按钮弹出表单，填写客户名、产品名、数量、金额、截止日期 |
| 订单看板 | 状态切换 | 下拉菜单改变订单状态，触发物料检查和工单生成 |

## 3. 核心流程

订单从创建到完成的全流程：
1. 运营者在订单看板新建订单 → 状态为"待处理"
2. 系统自动检查物料库存 → 不足则生成采购提醒
3. 运营者将订单状态改为"生产中" → 后端自动生成工单
4. 工单排程考虑物料可用性 → 物料不足则状态为"等待物料"
5. 物料补货后 → 工单状态变为"进行中"
6. 订单完成后状态变为"待发货" → 自动出现在发货计划
7. 确认发货 → 订单完成，Dashboard统计更新

## 4. 用户界面设计

### 4.1 设计风格
- 主色调：紫色系（#6366F1）和蓝色系（#3B82F6）
- 辅助色：绿色（#10B981）完成、红色（#EF4444）预警、黄色（#F59E0B）等待
- 布局：深色侧边栏（#1E293B）+ 浅色内容区（#F8F9FA）双栏布局
- 字体：系统无衬线字体，全局14px基准
- 卡片与交互元素：细微阴影（#CBD5E1 0 2px 6px）和圆角（8-12px）
- 状态切换与弹窗：0.2-0.3秒过渡动画（ease-out）

### 4.2 页面设计概览
| 页面名称 | 模块名称 | UI元素 |
|----------|----------|--------|
| 仪表盘 | 概览卡片 | 宽200px高120px圆角12px浅色渐变背景数字48px |
| 仪表盘 | 工单列表 | 每项高64px圆角8px深色#1E293B标签彩色Badge |
| 仪表盘 | 折线图 | Canvas宽100%高200px背景#F8FAFC线条#6366F1渐变填充 |
| 订单看板 | Kanban列 | 每列宽280px背景#F1F5F9订单卡片宽250px白底圆角8px阴影 |
| 订单看板 | 新建按钮 | 宽160px高48px圆角24px紫色渐变悬停1.05倍 |

### 4.3 响应式
- 桌面优先设计，最小宽度1024px
- 侧边栏固定240px宽度
- 内容区自适应剩余空间
