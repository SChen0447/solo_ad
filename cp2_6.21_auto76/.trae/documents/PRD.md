## 1. 产品概述

手工坊订单管理与生产排程应用是一款面向中小型手工作坊运营者的全流程管理工具，整合订单录入、物料库存、生产排程和发货计划，解决接单到出货之间的时间错配和库存积压问题。

- **目标用户**：中小型手工作坊运营者
- **核心价值**：实现从接单到出货的全流程可视化与自动化排程
- **解决问题**：订单与生产脱节、物料库存积压、排程混乱、发货延迟

## 2. 核心功能

### 2.1 功能模块

1. **仪表盘**：生产概览卡片、工单列表、7天趋势折线图
2. **订单看板**：Kanban风格订单状态管理、新建订单、状态流转
3. **物料库存**：物料表格展示、库存预警、补货操作
4. **生产工单**：工单列表、状态跟踪、详情查看
5. **发货计划**：时间线展示、发货确认

### 2.2 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|---------|---------|---------|
| 仪表盘 | 概览卡片 | 今日订单数、待开工工单、库存预警、待发货数统计卡片 |
| 仪表盘 | 工单列表 | 展示最近工单状态（等待物料/进行中/已完成） |
| 仪表盘 | 趋势图 | Canvas绘制最近7天订单量与完工量折线图 |
| 订单看板 | Kanban列 | 待处理/生产中/待发货/已完成四列状态流转 |
| 订单看板 | 新建订单 | 弹窗表单录入客户、产品、数量、金额、截止日期 |
| 物料库存 | 物料表格 | 展示物料名称、类别、库存、安全库存、单位、供应商 |
| 物料库存 | 补货功能 | 点击补货按钮增加库存，记录补货时间 |
| 发货计划 | 时间线 | 按日期垂直排列待发货订单，支持勾选确认发货 |

## 3. 核心流程

```mermaid
flowchart LR
    A["新建订单"] --> B["订单待处理"]
    B --> C["物料检查"]
    C -->|物料充足| D["生成工单"]
    C -->|物料不足| E["库存预警"]
    E --> F["补货操作"]
    F --> D
    D --> G["生产中"]
    G --> H["待发货"]
    H --> I["发货确认"]
    I --> J["已完成"]
```

用户新建订单后，系统自动检查物料库存。若物料充足则自动生成工单进入生产；若物料不足则触发预警，用户补货后工单开始生产。生产完成后订单转入待发货，用户确认发货后订单完成。

## 4. 用户界面设计

### 4.1 设计风格

- **主色调**：紫色系（#6366F1）、蓝色系（#3B82F6）
- **状态色**：绿色（#10B981）完成、红色（#EF4444）预警、黄色（#F59E0B）等待
- **布局**：左侧固定深色侧边栏（240px，#1E293B）+ 右侧浅色内容区（#F8F9FA）
- **卡片风格**：圆角8-12px，细微阴影（#CBD5E1 0 2px 6px）
- **字体**：系统无衬线字体，基准字号14px

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI元素 |
|---------|---------|-------|
| 仪表盘 | 概览卡片 | 200x120px卡片，浅色渐变背景，48px数字，圆角12px |
| 仪表盘 | 工单列表 | 64px高列表项，深色标签卡，彩色状态Badge |
| 仪表盘 | 趋势图 | Canvas折线图，区域渐变填充，悬停tooltip |
| 订单看板 | Kanban卡片 | 250px宽白色卡片，边框阴影，悬停放大1.03倍 |
| 订单看板 | 新建按钮 | 160x48px渐变色按钮，圆角24px，悬停放大1.05倍 |
| 物料库存 | 表格行 | 48px高，悬停背景变化，低库存行红色高亮 |
| 发货计划 | 时间线 | 圆点标记（12px直径），当前日期蓝色侧边条 |

### 4.3 动效设计

- 状态切换和弹窗过渡：0.2-0.3秒 ease-out
- 卡片展开收起：平滑过渡
- 列表行淡入淡出：opacity动画
- 发货确认后卡片：右滑消失0.5秒
- 库存预警数字：每秒闪烁一次，持续3秒

### 4.4 侧边栏导航

- 固定宽度240px，背景#1E293B
- 导航项白色文字，悬停背景#334155
- 激活项左侧紫色竖条（4px宽，80%高）+ 浅紫背景
- SVG图标（20x20px），分别对应订单、库存、工单、发货
- 库存图标带红色感叹号角标（12x12px）表示预警
