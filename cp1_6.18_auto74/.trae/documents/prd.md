## 1. 产品概述

碎片回合制RPG战斗应用，帮助玩家在通勤等零碎时间中快速体验完整的回合制战斗流程，解决传统RPG战斗节奏缓慢、等待时间长的痛点。面向喜爱RPG但时间碎片化的移动端和桌面端玩家。

## 2. 核心功能

### 2.1 用户角色
无角色区分，单玩家体验。

### 2.2 功能模块
1. **英雄选择页**：6选3组队、卡片翻转入场动画、技能冷却环形进度条
2. **战斗页**：自动回合战斗、手动技能释放、AI自动战斗、战斗日志打字机效果、实时战况动画
3. **结算页**：评分计算（S/A/B/C）、连胜特效、积分累计与解锁

### 2.3 页面详情
| 页面名称 | 模块名称 | 功能描述 |
|---------|---------|---------|
| 英雄选择页 | 角色卡片列表 | 展示6个预设英雄卡片（战士、法师、弓箭手、刺客、圣骑士、术士），每张显示HP/ATK/DEF和技能，点击选择/取消选择，最多3人 |
| 英雄选择页 | 翻转入场动画 | 角色卡片以3D翻转动画依次入场 |
| 英雄选择页 | 技能冷却进度条 | 头像底部环形进度条展示技能冷却状态 |
| 英雄选择页 | 开始战斗按钮 | 选满3人后激活，点击进入战斗 |
| 战斗页 | 英雄栏 | 显示我方3个英雄头像、血条、技能冷却脉冲光效 |
| 战斗页 | 敌方栏 | 显示敌方3个敌人头像、血条 |
| 战斗页 | 技能释放 | 点击英雄头像手动释放技能（消耗1回合冷却），或AI自动战斗 |
| 战斗页 | 战况动画 | 攻击屏幕震动、伤害数字缩放淡出、暴击红色闪电特效、死亡粒子消散 |
| 战斗页 | 战斗日志 | 右侧面板打字机效果逐行显示，毛玻璃背景；手机端隐藏改为弹窗 |
| 战斗页 | AI自动战斗 | 默认攻击低血量敌人，决策响应<50ms |
| 结算页 | 评分展示 | 根据剩余血量和回合数计算S/A/B/C评分 |
| 结算页 | 连胜特效 | 屏幕上方金色流光文字显示连胜次数 |
| 结算页 | 积分与解锁 | 累计积分解锁新角色和皮肤（变色特效） |

## 3. 核心流程

玩家打开应用 → 进入英雄选择界面 → 从6个角色中选择3人组队 → 点击开始战斗 → 进入战场，战斗以回合制自动进行 → 每回合可手动释放技能或让AI自动战斗 → 战斗日志实时滚动显示 → 战斗胜利/失败 → 显示评分和连胜奖励 → 可选择继续战斗或重新选人

## 4. 界面设计

### 4.1 设计风格
- 主色调：米黄 #F5E6C8（羊皮纸色）、深棕 #5D4037（边框装饰）、暗红 #8B0000（强调色）
- 战场背景：墨绿 #1A3A2A（深色森林色调）
- 按钮样式：圆角矩形，0.1s按压缩放效果
- 字体：MedievalSharp（标题）、Noto Serif SC（正文）
- 布局：手机竖屏居中、桌面端左右分栏
- 图标：扁平化几何符号（剑、法杖、弓、匕首、盾、骷髅）
- 卡片：圆角矩形+木纹边框装饰

### 4.2 页面设计概览
| 页面名称 | 模块名称 | UI元素 |
|---------|---------|--------|
| 英雄选择页 | 角色卡片列表 | 木纹边框圆角卡片、3D翻转动画、环形冷却进度条、几何技能图标 |
| 战斗页 | 战场区域 | 墨绿森林背景、英雄/敌人头像栏、血条、伤害数字浮动、粒子特效 |
| 战斗页 | 战斗日志面板 | 毛玻璃效果(backdrop-filter:blur(8px))、打字机文字效果 |
| 结算页 | 评分与奖励 | 金色流光文字、评分徽章、皮肤解锁预览 |

### 4.3 响应式设计
- 桌面优先设计，适配1024px以上宽度
- 手机端（400px以上）自动隐藏战斗日志，改为点击图标弹窗查看
- 所有交互元素支持触摸操作

### 4.4 性能要求
- 60FPS稳定帧率
- 技能特效粒子数量≤50个
- 战斗AI决策响应<50ms
- CSS transition + requestAnimationFrame驱动动画
