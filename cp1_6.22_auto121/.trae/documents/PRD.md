# 邻里技能交换与时间银行 - 产品需求文档 (PRD)

## 1. 项目概述

### 1.1 项目背景
小区里总有各种身怀绝技的邻居，有的人会修电脑、有的人会做蛋糕、有的人英语特别好，但大家都不知道彼此有什么特长，偶尔需要帮忙也只能花钱找外面的人。本项目旨在打造一个时间银行式的技能交换平台，让邻居们可以通过虚拟时间币进行技能交换，不花一分钱就能解决很多生活小麻烦，还能拉近邻里关系。

### 1.2 项目目标
- 建立邻里技能信息共享平台
- 实现时间币驱动的技能交换机制
- 促进社区互动，拉近邻里关系
- 提供便捷、可靠的技能预约与评价系统

### 1.3 目标用户
- 小区居民（技能提供者）
- 有技能需求的居民（服务使用者）
- 社区管理员

## 2. 核心功能需求

### 2.1 技能集市首页
- **布局形式**：技能卡片流展示
- **卡片规格**：宽320px、高240px，圆角20px，背景#f0fdf4，带8px浅绿阴影#86efac
- **卡片内容**：
  - 顶部：技能图标（Emoji）和标题
  - 中间：提供者圆形头像（直径64px）、昵称、信用评分（1-5星，金色#fbbf24）
  - 底部：服务耗时（如"需30分钟"）、可用时段标签（绿色"今日可约"/灰色"下周可约"）
- **交互效果**：鼠标悬停时卡片向上浮动10px，阴影增加至12px，过渡动画300ms ease
- **加载机制**：懒加载分页，每次显示12个技能卡片，滚动自动加载下一批
- **性能要求**：页面加载时间不超过2秒

### 2.2 技能详情页
- **上半部分左侧（提供者信息）**：
  - 详细个人信息
  - 可用日期周历：7个小圆点表示周一到周日，绿色#22c55e=有空、灰色#d1d5db=无、橙色#f97316=约满
  - 时间选择器：点击绿色日期弹出，下拉框圆角8px，白底#ffffff，浅灰边框#e2e8f0
- **上半部分右侧（评价列表）**：
  - 每条评价：圆角10px、背景#f8fafc卡片
  - 包含：评价者头像、评价内容、评分星级、评价时间
  - 排序：最新评价在最上方
- **下半部分（相关推荐）**：
  - 基于技能标签匹配，展示3个最相关的其他技能
  - 卡片滑动轮播，左右三角箭头按钮
  - 点击后平滑滑动一屏距离，动画600ms ease-in-out

### 2.3 个人空间（需登录）
- **顶部**：头像、大号昵称
- **Tab切换**：三个标签页
  - **我的技能**：列表显示自己发布的技能，右侧有编辑（蓝色#3b82f6）和删除（红色#ef4444）按钮
    - 编辑模态框：半透明黑色背景rgba(0,0,0,0.4)，居中白底面板宽500px
  - **我的预约**：时间线方式排列
    - 每条记录：日期、技能标题、提供者头像、联系状态按钮
    - 已完成服务：绿色对勾标识，可写评价
  - **我的时间币**：
    - 当前余额显示
    - 交易流水：类型"获得/支出"、对方头像、时间币数量、交易时刻
    - 默认显示最近20条，点击"加载更多"获取下20条

### 2.4 发布新技能表单
- **字段**：
  - 技能标题：输入框，圆角8px、白底#ffffff、浅灰边框#d1d5db、placeholder灰色#9ca3af
  - 技能简介：textarea，高度120px，同样样式
  - 耗时：数字输入，单位分钟，默认30
  - 可用时间段：周一到周日日期多选，方块宽40px高40px圆角8px，选中绿色#22c55e，未选中浅灰#e2e8f0
- **提交反馈**：点击发布后按钮变为"发布中..."，持续1.5秒后成功，跳转到我的技能Tab

### 2.5 预约功能
- **预约按钮**：宽180px高48px，渐变背景#22c55e→#16a34a，圆角32px，文字白色#ffffff字重600
- **成功场景**：时间币足够则立即扣减，弹出绿色圆角12px成功提示，1秒后自动消失
- **失败场景**：时间币不足则弹出红色圆角12px错误提示，底部显示"去赚取时间币"链接，跳转至技能集市
- **性能要求**：预约操作响应时间不超过500ms

## 3. UI/UX 设计规范

### 3.1 整体风格
- **风格定位**：清新自然风
- **主色调**：绿色系#22c55e
- **辅助色**：白色#ffffff、浅灰色#f8fafc
- **设计理念**：柔和、友好、社区感

### 3.2 布局规范
- **桌面端**：左右两栏结构，左栏70%主要内容，右栏30%侧边栏（用户信息汇总、最新动态流）
- **响应式断点**：
  - 1024px以下：右栏折叠至下方，变为全宽容器
  - 768px以下：卡片变为单列布局

### 3.3 动效规范
- **页面切换**：淡入过渡300ms ease，卡片从右向左滑动入场（偏移150px，持续400ms ease-out）
- **按钮交互**：圆角、柔和阴影
- **页面Title**：每个页面Title栏左侧有emoji小图标

## 4. 技术架构概览

### 4.1 前端技术栈
- React 18 + TypeScript
- Vite 构建工具
- React Router DOM 路由
- Axios HTTP客户端

### 4.2 后端技术栈
- Node.js + Express
- TypeScript
- CORS 跨域处理
- UUID 唯一标识生成
- 内存数据存储

### 4.3 数据模型
- **用户（User）**：id、昵称、头像、信用评分、时间币余额
- **技能（Skill）**：id、标题、简介、图标、耗时、提供者id、可用日期、标签
- **预约（Booking）**：id、技能id、预约者id、提供者id、日期、时间、状态、评价
- **交易（Transaction）**：id、类型（获得/支出）、用户id、对方id、数量、时间、关联预约id

## 5. 非功能需求

### 5.1 性能要求
- 页面加载时间 ≤ 2秒
- 预约操作响应时间 ≤ 500ms
- 图片懒加载
- 无限滚动分页加载

### 5.2 安全要求
- 用户身份验证
- 敏感操作防重复提交
- 输入数据校验

### 5.3 兼容性要求
- 支持主流现代浏览器（Chrome、Firefox、Safari、Edge）
- 响应式适配多端设备
