## 1. 产品概述

国际化输入格式验证与转换预览应用，帮助前端开发者在构建复杂表单时，实时验证用户输入的日期、时间、数字、货币格式是否符合国际标准（ISO-8601），并快速预览多语言环境下的格式转换效果。

- 面向前端开发者和国际化产品团队，解决多区域格式验证和预览的痛点
- 支持中文、英文、日文、阿拉伯文四种语言区域的实时转换预览，提升开发效率

## 2. 核心功能

### 2.1 功能模块

1. **输入面板**: 日期、时间、数字、货币四个独立输入区域，实时格式校验
2. **预览网格**: 四区域格式转换结果并排展示，实时更新
3. **状态管理**: 集中式状态管理，协调输入、校验、转换各模块通信
4. **格式转换引擎**: 基于 Intl API 的 ICU 规范格式转换
5. **输入解析引擎**: 智能输入模式识别和格式错误检测

### 2.2 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|---------|---------|---------|
| 主页面 | 顶部标题区 | 应用名称展示，品牌识别 |
| 主页面 | 输入面板 | 四个带浮动标签的输入框，实时校验状态提示，自动补全分隔符 |
| 主页面 | 预览网格 | 2x2 四区域格式预览面板，转换结果动态着色，悬停动效 |

## 3. 核心流程

用户在输入面板中输入数据 → 输入解析引擎实时识别输入类型并校验格式 → 状态管理器更新输入值和校验状态 → 格式转换引擎根据四种区域设置转换格式 → 预览网格实时展示转换结果。

```mermaid

    A["用户输入"] --> B["inputParser 解析校验"]
    B --> C{"格式是否有效"}
    C -->|是| D["formatStore 更新状态"]
    C -->|否| E["标记错误状态和错误类型"]
    E --> D
    D --> F["localeConverter 多区域转换"]
    F --> G["PreviewGrid 渲染预览结果"]
    D --> G
```

## 4. 用户界面设计

### 4.1 设计风格

- **主色调**: 深蓝 #2c3e50（标题栏）、浅蓝 #a0c4ff（强调色）
- **功能色**: 绿色 #27ae60（正确）、红色 #e74c3c（错误）
- **中性色**: 深灰 #333、浅灰 #999、极浅灰 #f9f9f9、白色 #ffffff
- **字体**: 系统无衬线字体，标题 28px 粗体，正文 16px
- **布局**: 居中单栏，左右两栏（输入+预览），响应式堆叠
- **动效**: 平滑过渡动画 0.2s ease，输入框聚焦标签上移，悬停边框变化

### 4.2 页面设计概述

| 页面名称 | 模块名称 | UI 元素 |
|---------|---------|---------|
| 主页面 | 顶部标题区 | 高度 80px，深蓝背景，白色粗体字，底部 2px 阴影 |
| 主页面 | 输入面板 | 浅灰背景 #f9f9f9，圆角 10px，内边距 24px，浮动标签输入框，实时校验提示 |
| 主页面 | 预览网格 | 2x2 网格 620px 总宽，白色卡片，浅灰边框，4px 内阴影，悬停淡蓝边框 |

### 4.3 响应式

- 桌面端（≥800px）: 左右两栏布局，输入面板宽 380px，间距 40px
- 移动端（<800px）: 上下堆叠布局，所有间距减半
- 输入框和预览区域均适配容器宽度
