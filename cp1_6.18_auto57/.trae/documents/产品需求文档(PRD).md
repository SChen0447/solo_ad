## 1. 产品概述

食谱分享与营养换算器是一款让用户创建、分享个人食谱并自动计算营养信息的Web应用。用户可以录入食材及其营养参数，系统自动计算总热量、蛋白质、脂肪和碳水化合物含量，并支持按份数动态缩放。

- **目标用户**：美食爱好者、健身人士、营养师、家庭主妇
- **核心价值**：简化食谱营养计算流程，直观可视化营养成分，支持灵活的份数调整

## 2. 核心功能

### 2.1 功能模块

1. **首页（HomePage）**：食谱卡片网格展示、搜索过滤、创建新食谱浮层表单、收藏功能
2. **食谱详情页（RecipeDetailPage）**：完整做法展示、配料表、营养计算器、份数缩放、复制分享链接

### 2.2 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|---------|---------|---------|
| 首页 | 搜索框 | 按标题关键字搜索食谱，0.3秒渐隐渐现过滤动画 |
| 首页 | 食谱卡片网格 | 响应式grid布局（桌面4列/平板2列/手机1列），展示标题、图片、营养摘要 |
| 首页 | 创建食谱浮层 | 淡入动画弹出表单，包含标题、图片URL、配料列表、步骤描述 |
| 首页 | 收藏按钮 | 心形按钮切换实心/空心，弹出"已收藏"toast提示 |
| 食谱详情页 | 配料与步骤 | 完整展示食材用量和做法步骤 |
| 食谱详情页 | 营养计算器 | 根据份数自动计算并展示热量、蛋白质、脂肪、碳水 |
| 食谱详情页 | 份数选择器 | 1-10份加减按钮，数字平滑过渡动画 |
| 食谱详情页 | 复制链接按钮 | 复制当前URL，显示"链接已复制"微提示 |

## 3. 核心流程

用户打开首页浏览食谱卡片 → 通过搜索框筛选食谱 → 点击卡片进入详情页查看完整配方 → 调整份数查看动态营养数据 → 复制链接分享 / 返回首页收藏或创建新食谱

## 4. 用户界面设计

### 4.1 设计风格

- **主色调**：番茄红 `#E53935`
- **背景色**：浅米色 `#F9F6F0`
- **文字色**：深棕色 `#3E2723`
- **卡片样式**：白色圆角矩形（border-radius: 12px），缓升阴影，hover时阴影加重并上移2px
- **按钮风格**：圆角按钮，番茄红主色，0.2s ease-in-out过渡
- **字体**：Google Fonts - Lora（标题衬线体）+ Nunito Sans（正文字体）
- **图标**：Font Awesome 6 图标库

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI元素 |
|---------|---------|---------|
| 首页 | 顶部导航 | 暖色系背景，居中应用标题，右侧创建食谱按钮 |
| 首页 | 搜索框 | 圆角输入框，左侧搜索图标，placeholder浅灰色 |
| 首页 | 食谱卡片 | 白色圆角卡片，封面图、标题、营养条、收藏按钮 |
| 首页 | 浮层表单 | 半透明遮罩，中心面板淡入，表单行垂直布局 |
| 首页 | 空状态 | 居中插画 + 提示文字 |
| 食谱详情页 | 标题区 | 大标题、复制链接按钮、收藏按钮 |
| 食谱详情页 | 营养条 | 四段渐变色彩条（红/蓝/黄/绿），hover放大显示数值 |
| 食谱详情页 | 份数控制 | 加减按钮 + 中央数字，过渡动画 |
| 食谱详情页 | 配料列表 | 每行展示食材名称、数量、单位、营养数据 |

### 4.3 响应式设计

- **桌面端**（≥1200px）：4列grid布局
- **平板端**（768px-1199px）：2列grid布局
- **移动端**（<768px）：1列grid布局，搜索框全宽

### 4.4 动画与交互

- 食谱卡片hover：上移2px + 阴影加深，0.2s ease
- 搜索过滤：0.3s渐隐渐现
- 浮层表单：淡入 + 轻微缩放
- 新卡片插入：缩放动画（scale 0.9 → 1）
- 营养数值变化：数字递增/递减平滑过渡，颜色微调
- 份数按钮：点击缩放反馈
- toast提示：上滑出现，自动淡出
