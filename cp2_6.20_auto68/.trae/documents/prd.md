## 1. 产品概述

声音解谜2D平台跳跃游戏——玩家通过麦克风发出不同频率和时长的声音来控制平台升降、移动障碍物或触发机关，最终到达终点。目标用户为独立游戏爱好者和音游玩家，核心价值在于"以声控解谜"的独特玩法与赛博朋克视听体验。

## 2. 核心功能

### 2.1 功能模块

1. **校准界面**：麦克风权限获取、5秒"啊"声校准、声波可视化圆环与音量条
2. **关卡选择界面**：三张关卡卡片、解锁进度管理
3. **游戏主界面**：玩家控制、声控机关交互、碰撞检测、声波可视化
4. **通关界面**：通关提示、下一关按钮
5. **重试机制**：掉落或R键重置关卡

## 3. 核心流程

1. 启动游戏 → 请求麦克风权限 → 进入校准界面
2. 校准5秒"啊"声 → 检测平均频率 → 归零校准 → 自动进入关卡选择
3. 关卡选择 → 点击已解锁卡片 → 进入游戏
4. 游戏中：键盘控制移动+声音控制机关 → 到达终点 → 通关
5. 通关 → 显示"恭喜通关" → 下一关/返回关卡选择
6. 失败（掉落/R键） → 重置关卡

## 4. 用户界面设计

### 4.1 设计风格

- **主色调**：深蓝（#0B0C10）+ 青色（#66FCF1）+ 紫色渐变
- **辅助色**：暗红（#C5C6C7）、橙色（#F3A183）、亮黄（#FFD700）
- **按钮风格**：圆角8px，渐变背景（#45A29E → #66FCF1），hover缩放1.05+光晕
- **字体**：标题青色（#66FCF1），正文白色（#FFFFFF）
- **布局**：全屏Canvas，16:9自适应比例
- **背景**：径向渐变（中心#1F2833→边缘#0B0C10），星星粒子

### 4.2 页面设计概述

| 页面名称 | 模块名称 | UI元素 |
|---------|---------|--------|
| 校准界面 | 声波圆环 | 深蓝→蓝紫径向渐变背景，中心圆环50-120px动态缩放，蓝→红0.2秒渐变 |
| 校准界面 | 音量条 | 200px宽矩形条，半透明背景，高度随音量变化 |
| 关卡选择 | 卡片 | 200x250px，圆角12px，深灰#2C2F33背景+内阴影，解锁亮色渐变 |
| 游戏主界面 | 声波可视化 | 200x80px，圆角8px，半透明深色#1F2833背景，青→紫渐变线条 |
| 游戏主界面 | 玩家 | 20px直径圆形，#FFD700，25px半径光晕透明度0.3 |
| 游戏主界面 | 音控平台 | 200x20px，青→蓝渐变#45A29E→#66FCF1 |
| 游戏主界面 | 音控门 | 25x100px，暗红#C5C6C7，水平滑动0.3秒 |
| 游戏主界面 | 音控方块 | 40x40px，橙色#F3A183 |
| 通关界面 | 文字+按钮 | 48px #66FCF1标题，渐变按钮圆角8px |

### 4.3 响应式适配

- 画布自适应窗口大小，保持16:9比例
- Canvas内容按虚拟分辨率（1920x1080）缩放绘制
- 所有UI元素基于虚拟坐标系统定位

### 4.4 动效规范

- 界面切换：淡入淡出0.2秒
- 按钮hover：缩放1.05倍+光晕，过渡0.2秒
- 终点星形：0.5秒闪烁周期
- 星星粒子：随机分布，缓慢闪烁
