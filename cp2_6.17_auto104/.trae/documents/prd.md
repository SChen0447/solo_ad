## 1. 产品概述

音乐节现场观众互动打分与实时热力看板应用，旨在替代低效的纸质投票方式，为音乐节提供数字化实时评分与评论系统。观众可通过手机对舞台表演进行1-5星评分和文字评论，主办方通过大屏幕实时展示热力分布和观众口碑，增强现场互动氛围和视觉冲击力。

- 目标用户：音乐节现场观众（投票者）、主办方管理员（管理者）
- 核心价值：实时数据可视化、零延迟互动体验、替代传统纸质投票

## 2. 核心功能

### 2.1 用户角色

| 角色 | 注册方式 | 核心权限 |
|------|----------|----------|
| 观众 | 无需注册 | 对舞台打分、提交评论 |
| 管理员 | 无需注册（直接访问管理面板） | 创建舞台、控制投票开关、查看实时数据总览 |

### 2.2 功能模块

1. **看板页面（LiveDashboard）**：热力地图区域、滚动评论墙
2. **投票功能（VotingPanel）**：舞台编号输入、座位号输入、星形评分、文字评论、验证码
3. **后台管理面板（AdminPanel）**：舞台创建、投票开关控制、实时评分总览表格

### 2.3 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|
| 看板页面 | 热力地图区域 | 网格布局展示舞台区块，根据平均分渐变颜色（红→黄→绿），悬浮放大效果，每5秒轮询更新 |
| 看板页面 | 滚动评论墙 | 最新评论从上到下排列，每条含头像/昵称/评论/星级，30秒自动滚动，新评论淡入动画，每3秒轮询 |
| 投票功能 | 评分表单 | 输入6位舞台编号+座位号，1-5星点击评分，200字以内评论，4位验证码验证 |
| 管理面板 | 舞台创建 | 输入名称/描述/起始时间，创建后表单清空并刷新列表，最多10个舞台 |
| 管理面板 | 投票开关 | 圆角矩形按钮切换投票开启/关闭，即时变化无延迟 |
| 管理面板 | 评分总览表格 | 展示舞台名称/平均分/投票人数/最高分 |

## 3. 核心流程

**观众投票流程**：观众输入舞台编号和座位号 → 选择1-5星评分 → 输入文字评论 → 填写验证码 → 提交至后端 → 数据更新至热力图和评论墙

**管理员流程**：管理员访问后台面板 → 创建舞台（名称/描述/时间） → 开启投票开关 → 观众开始投票 → 实时查看评分总览

## 4. 用户界面设计

### 4.1 设计风格

- **看板页面**：深色沉浸风格，深蓝到深紫渐变背景（#0a0e27到#1a1040），霓虹感配色
- **管理面板**：浅色专业风格（#f0f2f5），清爽表格布局
- **主色调**：热力图渐变（红#ff4d4f→黄#fadb14→绿#52c41a）
- **按钮风格**：圆角6px，悬停缩放1.05倍
- **字体**：看板用无衬线白色文字，管理面板用深色文字
- **布局**：看板为上下结构（热力图+评论墙），管理面板为表格+表单布局

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI元素 |
|----------|----------|--------|
| 看板页面 | 热力地图 | 网格布局，圆角12px区块，渐变色背景，悬浮放大1.05倍+阴影过渡0.2s，舞台名称+平均分+投票人数 |
| 看板页面 | 滚动评论墙 | 磨砂玻璃背景(blur:8px, rgba(255,255,255,0.05))，圆形渐变头像32px，白色12px昵称，白色14px评论，红色星星，淡入动画0.3s ease |
| 投票功能 | 评分表单 | 深色半透明卡片，黄色填充星星，输入框，验证码区域 |
| 管理面板 | 舞台创建表单 | 浅色卡片，输入框，提交按钮 |
| 管理面板 | 投票开关按钮 | 圆角矩形，开启绿色#52c41a，关闭灰色#d9d9d9 |
| 管理面板 | 评分总览表格 | 白色背景，行悬浮高亮#e6f7ff |

### 4.3 响应式设计

- 桌面优先设计
- 宽度<768px时：热力图变为纵向堆叠，评论墙变为可滑动横向列表
- 所有页面支持响应式布局

## 5. 性能约束

- 页面初始加载 ≤ 2秒
- 热力图仅更新变化区块DOM，避免全量重绘
- 评论墙≤200条时帧率>50fps
- 轮询间隔：热力图5秒，评论墙3秒
