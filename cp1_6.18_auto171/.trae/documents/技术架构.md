# 创意众包与灵感投票应用 - 技术架构文档

## 1. 架构设计原则

### 1.1 分层架构
采用关注点分离的分层架构设计：
- **数据层**：Zustand状态管理，负责数据增删改查
- **业务逻辑层**：VoteEngine投票与排名引擎，独立计算逻辑
- **UI层**：React组件，纯渲染逻辑

### 1.2 核心原则
- 数据管理与UI渲染分离
- 投票逻辑与状态管理分离
- 单向数据流
- 类型安全（TypeScript严格模式）

## 4. 核心模块设计

### 4.1 类型定义 (types.ts)

```typescript
// 创意类别
type Category = 'all' | 'tech' | 'art' | 'life' | 'business';

// 用户类型
interface User {
  id: string;
  name: string;
  avatar?: string;
}

// 评论类型
interface Comment {
  id: string;
  creativeId: string;
  userId: string;
  userName: string;
  content: string;
  createdAt: number;
}

// 投票记录
interface Vote {
  creativeId: string;
  userId: string;
  votedAt: number;
}

// 创意数据
interface Creative {
  id: string;
  title: string;
  description: string;
  category: Exclude<Category, 'all'>;
  authorId: string;
  authorName: string;
  createdAt: number;
  votes: number;
  comments: Comment[];
}

// 排行榜条目
interface LeaderboardItem {
  creative: Creative;
  rank: number;
  previousRank: number | null;
  trend: 'up' | 'down' | 'stable' | 'new';
  hotScore: number;
}
```

### 4.2 状态管理 (dataStore.ts)

**职责**：
- 创意数据的增删改查
- 投票记录管理
- 评论管理
- 类别筛选状态
- 提供订阅接口

**核心方法**：
```typescript
- addCreative(creative: Omit<Creative, 'id' | 'createdAt' | 'votes' | 'comments'>): void
- deleteCreative(id: string): void
- addVote(creativeId: string, userId: string): boolean
- removeVote(creativeId: string, userId: string): boolean
- hasVoted(creativeId: string, userId: string): boolean
- addComment(comment: Omit<Comment, 'id' | 'createdAt'>): void
- setCategory(category: Category): void
- getCreativesByCategory(): Creative[]
- subscribe(listener: (state: DataStore) => void): () => void
```

### 4.3 投票与排名引擎 (VoteEngine.ts)

**职责**：
- 计算热度分数（结合投票数和时间衰减）
- 计算趋势变化（排名升降）
- 生成排行榜数据
- 格式化投票数显示

**核心方法**：
```typescript
- getHotScore(creative: Creative): number
- getLeaderboard(category?: Category, limit?: number): LeaderboardItem[]
- formatVoteCount(votes: number): string
- getTrend(currentRank: number, previousRank: number | null): 'up' | 'down' | 'stable' | 'new'
```

**热度算法**：
```
hotScore = votes * 1000 / (hoursSinceCreation + 2)^1.5
```

## 5. 组件设计

### 5.1 CreativeCard 创意卡片

**Props**：
- creative: Creative
- onVote: () => void
- hasVoted: boolean
- onClick?: () => void

**状态**：
- isAnimating: boolean（投票动画）

**动画**：
- 进入：从上方滑入，0.4s弹性缓出
- 投票：填充+缩放，0.2s

### 5.2 Leaderboard 排行榜

**Props**：
- items: LeaderboardItem[]
- onItemClick: (creativeId: string) => void

**特性**：
- 每5秒自动刷新
- 排名变化平滑过渡（FLIP动画）
- 前三名特殊颜色

### 5.3 App 主应用

**路由**：
- `/`：首页（创意列表 + 排行榜）
- `/creative/:id`：创意详情页

**布局**：
- 桌面端：左右两栏
- 移动端：单列，排行榜置顶

## 6. 数据流

```
用户操作 → UI组件 → dataStore → VoteEngine → UI更新
    ↓                                        ↑
    └──────────── 订阅机制 ───────────────────┘
```

**投票流程**：
1. 用户点击投票按钮
2. CreativeCard 调用 dataStore.addVote()
3. dataStore 更新投票数和投票记录
4. Zustand 触发状态更新
5. VoteEngine 重新计算排行榜
6. UI 组件响应式更新

---

## 7. 性能优化

### 7.1 列表渲染
- 使用 React.memo 包裹 CreativeCard
- 使用稳定的 key（创意ID）
- 按需渲染描述片段

### 7.2 排行榜刷新
- 5秒轮询，使用 requestAnimationFrame 平滑过渡
- 防抖处理避免频繁重渲染
- 使用 FLIP 技术实现位置动画

### 7.3 状态更新
- Zustand 自动批处理
- 选择器模式避免不必要重渲染
- 投票操作内存级响应，< 200ms
