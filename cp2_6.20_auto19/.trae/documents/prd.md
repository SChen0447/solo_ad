## 1. 产品概述

动物领养信息管理应用——为小型动物收容所打造的全流程数字化领养平台，解决传统纸质登记效率低下、信息不透明等问题，帮助潜在领养人实时浏览可领养动物信息，在线提交领养申请，管理员高效审批，缩短动物等待领养的时间。

- 目标用户：潜在领养人（浏览动物、提交申请）、收容所管理员（审批申请、发布公告）
- 核心价值：提升领养流程效率，减少动物等待时间，实现信息透明化

## 2. 核心功能

### 2.1 用户角色

| 角色 | 注册方式 | 核心权限 |
|------|----------|----------|
| 潜在领养人 | 无需注册 | 浏览动物列表、查看动物详情、提交领养申请 |
| 管理员 | 通过 /admin 路径直接访问 | 审批/拒绝领养申请、查看所有申请记录 |

### 2.2 功能模块

1. **首页**：公告滚动条、动物卡片瀑布流列表、动物详情侧滑面板、领养申请弹窗
2. **管理后台页**：领养申请列表、审批操作、卡片翻转动画

### 2.3 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|
| 首页 | 公告滚动条 | 顶部水平滚动显示收容所动态，悬停暂停 |
| 首页 | 动物卡片列表 | 瀑布流布局展示动物卡片，含照片占位图、名字、品种、年龄、性格标签，悬停上浮4px |
| 首页 | 动物详情面板 | 点击卡片后从右侧滑入（0.3s），显示动物故事和健康状况，含"申请领养"按钮 |
| 首页 | 领养申请弹窗 | 从中心缩放出现（0.2s），含姓名/电话/住址/领养理由表单，提交后显示反馈动画 |
| 管理后台页 | 申请列表 | 表格展示所有领养申请，含申请人信息/动物名称/申请时间/状态标签 |
| 管理后台页 | 审批操作 | 通过/拒绝按钮，点击后卡片翻转（0.4s），显示审核结果和时间 |

## 3. 核心流程

1. **浏览领养流程**：用户进入首页 → 查看公告滚动条 → 浏览动物卡片列表 → 点击卡片查看详情 → 点击"申请领养" → 填写申请表单 → 提交 → 等待审核
2. **管理员审批流程**：管理员进入 /admin → 查看待审批申请 → 点击通过/拒绝 → 卡片翻转显示结果

## 4. 用户界面设计

### 4.1 设计风格

- 主色调：#F5A623（温暖橙色）+ #4A90D9（柔和蓝色）
- 背景色：#FFF8F0（浅米色）
- 卡片/弹窗：白色背景，圆角16px，多层柔和阴影
- 按钮：渐变背景（主色到同色系深色），悬停亮度提升10%，点击缩小0.95倍
- 字体：深灰色#333，搭配温暖友好的显示字体
- 导航栏：固定顶部，毛玻璃效果（backdrop-filter: blur(10px)）
- 图标：使用 lucide-react 图标库

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI元素 |
|----------|----------|--------|
| 首页 | 公告滚动条 | 全宽横条，浅橙色背景，文字从右向左滚动，悬停暂停 |
| 首页 | 动物卡片 | 白色圆角卡片，纯色背景+SVG动物剪影，3列瀑布流，悬停上浮+阴影加深 |
| 首页 | 动物详情面板 | 右侧滑入面板，浅米色背景，动物故事+健康状况，底部"申请领养"按钮 |
| 首页 | 领养申请弹窗 | 居中弹窗，半透明遮罩，缩放出现动画，4个表单字段，提交按钮带加载动画 |
| 管理后台页 | 申请列表 | 表格布局，状态标签（黄/绿/红），通过/拒绝按钮 |
| 管理后台页 | 审批卡片 | 翻转动画，正面显示申请信息，背面显示审核结果 |

### 4.3 响应式适配

- 桌面端（>1024px）：3列瀑布流，完整导航栏
- 平板端（768-1024px）：2列瀑布流，简化导航
- 手机端（<768px）：1列瀑布流，汉堡菜单（点击从顶部向下展开，菜单项依次出现间隔0.05s），触摸友好（点击区域≥44px）

### 4.4 动效规范

- 卡片悬停：上浮4px + 阴影加深，过渡0.2s
- 详情面板：从右侧滑入，持续0.3s
- 领养弹窗：从中心缩放出现，持续0.2s
- 提交按钮：加载旋转动画0.3s
- 反馈消息：2s后淡出（透明度1→0，0.5s）
- 审批卡片翻转：持续0.4s
- 页面切换：淡入淡出，持续0.2s
- 骨架屏：灰色闪烁动画，持续1.5s
- 所有动画使用 CSS transitions/animations 驱动
